# Security Policy

## Supported Versions

We actively maintain the latest version of this project. Security updates are applied to the following versions:

| Version | Supported          |
|--------|--------------------|
| Latest | ✅ Yes             |
| Older versions | ❌ No      |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### Please do NOT:
- Open a public issue
- Share the vulnerability publicly before it is resolved

### Please DO:
- Contact us directly via email: **security@example.com**
- Include as much detail as possible:
  - Description of the vulnerability
  - Steps to reproduce
  - Potential impact
  - Any proof of concept (if available)

## Response Process

Once a vulnerability is reported:

1. We will acknowledge receipt within 48 hours
2. We will investigate and validate the issue
3. We will work on a fix if necessary
4. We will notify the reporter once the issue is resolved
5. A security patch will be released if applicable

## Disclosure Policy

- We follow responsible disclosure practices
- Security issues will be addressed before public disclosure
- Credit may be given to reporters (if desired)

## Scope

This security policy applies to:
- Source code in this repository
- Official releases and builds
- Related services maintained by the project team

## Security Best Practices

When using this project, we recommend:
- Keeping dependencies up to date
- Using environment variables for sensitive data
- Avoiding hardcoding secrets in the codebase
- Enabling HTTPS in production environments
- Restricting access using proper authentication and authorization

For any security-related questions or concerns, contact:
